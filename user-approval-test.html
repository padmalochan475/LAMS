üìã User Approval Workflow Overview

This page tests the complete user approval system in LAMS. Here's how it works:

Step 1: New User Signs In
  • User clicks "Sign in with Google"
  • Google OAuth provides user credentials
  • System checks if user email matches admin email
  • If not admin, checks if user is in approved list
Step 2: User Added to Pending List
  • If user is not approved, they're added to pending users list
  • User data is immediately synced to Google Drive
  • Pending count is updated in admin UI
  • User sees message: "Access request sent to admin"
Step 3: Admin Reviews Pending Users
  • Admin sees pending users count in navigation
  • Admin opens "Manage Users" panel
  • All pending users are displayed with photos, names, emails
  • Admin can approve or reject each user
Step 4: User Approval/Rejection
  • Admin clicks "Approve" ‚Üí User moved to approved list
  • Admin clicks "Reject" ‚Üí User removed from pending list
  • Changes are immediately synced to Google Drive
  • All devices get updated user lists
